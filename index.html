  • 使用 React.memo 缓存纯组件。

  • 合理设置 key，避免不必要的 diff。

React Hooks 深入解析

\n+## 常用 Hooks

  • useState：管理本地状态。
  • useEffect：副作用处理（订阅/回收、数据请求）。
  • useMemo：记忆化计算，避免昂贵计算重复执行。
  • useCallback：记忆化函数，降低子组件不必要渲染。
  • useRef：持久化引用，用于 DOM 或保存可变值。
    \n+## useEffect 示例 \n+## 自定义 Hook \n+## 注意事项
  • 仅在函数最外层调用 Hooks；不要在循环、条件、嵌套函数中调用。
  • 所有依赖必须在依赖数组中声明，保证一致性与可维护性。

React 基础与组件化

\n+## 核心概念

  • 组件：UI 的最小单元，可复用与组合。
  • JSX：声明式语法糖，最终编译为 React.createElement。
  • Props：组件的输入参数，外部不可变。
  • State：组件内部状态，驱动 UI 变化。
    \n+## 函数组件示例 \n+## 受控与非受控组件
  • 受控：状态由 React 管理（如 <input value={v} onChange={...}>）。
  • 非受控：通过 ref 直接读写 DOM 值。
    \n+## 组件组织建议
  • 单一职责与清晰边界；组合胜过继承。
  • 提取通用组件与样式，避免重复实现。

Webpack 构建性能优化实践

\n+## 性能优化方向总览

  • 缓存：cache.type = 'filesystem'、babel-loader 的 cacheDirectory。
  • 并行：thread-loader 或 esbuild-loader/swc-loader。
  • 代码分割：optimization.splitChunks 与 dynamic import。
  • 体积压缩：TerserPlugin、CssMinimizerPlugin、图片压缩。
    \n+## 文件系统缓存 \n+## 代码分割示例 \n+## 图片与静态资源优化
  • 使用内置 asset 模式，按阈值自动转为 base64 或文件。
  • 在生产环境开启 image-minimizer-webpack-plugin；也可在 CI 中预处理图片。
    \n+## 构建速度排查
  • 开启 stats 或使用 webpack-bundle-analyzer 分析包体与模块依赖。
  • 关注长耗时 loader/plugin，按需替换为更快实现（如 esbuild/swc）。

Webpack Loader 与 Plugin 详解

\n+## Loader 与 Plugin 的区别

  • Loader：在模块解析阶段处理资源文件，将非 JS 资源转换为可打包的模块。例如：babel-loader、css-loader、file-loader。
  • Plugin：在构建生命周期的各个阶段插入逻辑，扩展打包能力。例如：HtmlWebpackPlugin、DefinePlugin、MiniCssExtractPlugin。
    \n+## 常见 Loader 示例 \n+## 常见 Plugin 示例 \n+## 选择与调优建议
  • 优先使用内置 asset 模式处理静态资源，减少额外 loader。
  • Plugin 过多会增加构建时间，按需启用并注意顺序与版本兼容。
  • 对于大型项目，结合 cache、thread-loader 与 babel 的 cacheDirectory 提升性能。

Webpack 入门与核心概念

Webpack 是什么

Webpack 是一个打包工具，核心目标是将模块化的资源（JS/CSS/图片等）通过依赖图打包输出，便于在浏览器或 Node 环境中高效加载。

核心概念

  • 入口（entry）：构建依赖图的起点。
  • 输出（output）：打包后文件的输出位置与命名规则。
  • Loader：处理非 JS 资源（如 CSS、图片、TS、Babel 等）。
  • Plugin：扩展打包能力（如生成 HTML、压缩代码、清理目录等）。
  • 模式（mode）：development 或 production，影响默认优化策略。

常见命令

  • webpack：进行一次构建（需安装 webpack 与 webpack-cli）。
  • webpack serve：启动开发服务器（需安装 webpack-dev-server）。

何时选择 Webpack

  • 需要成熟生态与细粒度可控的插件体系时。
  • 面向大型前端项目的稳定构建方案。
